# CLAUDE.md

SpeakIn：多對多即時串流語音翻譯管線（STT → MT → TTS）。

開工先讀 **《規則.md》**（架構導覽、事件流不變量、CI 陷阱）；
操作指令查 **《指令.md》**（啟動/測試/通道 runbook）；
**《結構及技術棧.md》** 是架構與 Phase 進度的唯一真相來源。

## 天條（違反會弄壞別人的東西，或讓下游全錯）

1. **`~/.local` 是 MISA 專案的環境，嚴禁升級/變動任何套件**（壞了只能同版本重裝）。
   SpeakIn 一律用獨立 venv（repo `.venv` → 本機 NVMe symlink）。
2. **`/nas-data/allen` 與本 repo 以外的 NAS 目錄是別人的，不能動。**
3. NAS 只當倉庫（寫入僅 ~63MB/s）：venv、模型熱快取、NATS 資料一律放本機 NVMe。
4. `schemas/events.py` 是全系統唯一合約：只准加欄位；事件流不變量
   （《規則.md》）不可破，topic 字串不可手拼。
5. 完成 Phase 項目或改動架構決策後，必須同步更新《結構及技術棧.md》（進度紀錄）
   與《代辦.md》（任務勾選/增刪）。架構與任務分離：前者不放待辦清單。
