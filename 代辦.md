# SpeakIn 代辦

> 目前狀態：Phase 4 剛完成 TTS worker 以 LiveKit 虛擬參與者發軌。
> 目標：先補齊多對多真實迴路的結構地基，再進 Phase 5 高扇出與插件驗證。

## P0：Phase 4 完成前必做

- [ ] **LiveKit mic → STT ingress**
  - viewer 發布麥克風後，STT worker 需從 LiveKit room 訂閱講者音軌，而不是只靠音檔 replay。
  - 輸出仍維持 `segment.stt`，一人一軌對應一個 `speaker_id`。

- [ ] **mt/tts 消費併發化，消除 head-of-line blocking**
  - 目前 NATS callback 等推論完成才返回，多講者會互相阻塞。
  - 改成 callback 只派工：per-speaker queue 保序，跨 speaker 並行。
  - vLLM 交給 continuous batching；TTS 保留 `Semaphore(2)` 作為 GPU 上限。

- [ ] **viewer 完整閉環**
  - 發布麥克風。
  - 選擇訂閱語言。
  - 訂閱 `text.{lang}` 顯示 partial/final 字幕。
  - 訂閱 LiveKit TTS 音軌，支援依 `speaker_id/lang` 選軌。

- [ ] **floor control v1**
  - 跟隨 active speaker。
  - 耳機保持單軌播放，字幕允許多人並行。
  - 發布 `control.floor`，viewer/audience 只依 control event 做 UI/音軌策略。

- [ ] **全鏈路 trace 可見**
  - viewer 或最小 dashboard 顯示每段 `stt → mt → tts` trace。
  - Phase 4 退出條件不只「能聽到」，還要能定位延遲瓶頸。

## P1：Phase 5 前結構補強

- [ ] **runner 拓撲接線實作**
  - 文件第 6 節承諾 `after:` 可自動改寫 topic，但目前 runner 只啟動 stage。
  - 需由 runner 注入 `input_subjects` / `output_topics`，stage 不應 hardcode 全域 topic。
  - 完成後才能真正做到 `plugins/glossary_corrector` 僅改 YAML 插入。

- [ ] **NATS JetStream 分層**
  - partial 可維持 best effort。
  - final、tts meta、control 建議走 JetStream durable stream。
  - 需要 ack、durable consumer、session replay，支援外部應用補讀。

- [ ] **Control plane 最小落地**
  - `session_manager` 維護 participant、speaker_id、selected_lang、active_speaker、lang_subscribers。
  - TTS 惰性啟動依據 `lang_subscribers`，未訂閱語言不合成。

- [ ] **ControlEvent 型別收斂**
  - 目前 `payload` 是自由 dict。
  - 建議保留 schema 擴充彈性，但在程式內用 `FloorPayload` / `SubsPayload` 驗證關鍵 control event。

## P2：Phase 5 驗證項

- [ ] **TTS 惰性啟動壓測**
  - 模擬 100+ 訂閱端。
  - 驗證聽眾數增加時 GPU 負載不隨之增加。

- [ ] **插件插拔實證**
  - 實作 `plugins/glossary_corrector`。
  - 插在 `stt → mt` 之間，只改 pipeline YAML，不改 core/stages。

- [ ] **Gateway**
  - WS 推逐字稿。
  - REST 收外部文字，轉成虛擬講者並進 TTS 房間。

- [ ] **stage-display / audience-app**
  - stage-display：大屏字幕。
  - audience-app：選語言、耳機、講稿。

## 架構原則備忘

- `SegmentEvent` 是核心合約，stage 之間不直接依賴彼此。
- `segment_id` 必須貫穿 `stt → mt → tts`，字幕、講稿、語音都靠它對齊。
- TTS 只消費 final；字幕可消費 partial。
- 成本模型要守住：計算成本跟 `同時開口人數 × 實際訂閱語言數` 相關，不跟聽眾人數線性成長。
